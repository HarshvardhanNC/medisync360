# 🛡️ Insurance Claim Intelligence System — MediSync360

A standalone Python Flask micro-service + Next.js UI that predicts insurance claim outcomes using a **rule engine + Logistic Regression ML model**.

---

## 📁 File Structure

```
insurance_ai/
├── app.py              ← Flask REST API  (port 5001)
├── model.py            ← ML model training & inference
├── rules.py            ← Hardcoded insurer policy rules
├── data_generator.py   ← Synthetic 500-row dataset generator
├── requirements.txt    ← Python dependencies
├── claim_model.pkl     ← Saved model (auto-generated on first run)
└── README.md

frontend/src/app/claim-intelligence/
└── page.tsx            ← Next.js UI page (route: /claim-intelligence)
```

---

## ⚡ Quick Start

### 1. Install Python Dependencies
```bash
cd insurance_ai
pip install -r requirements.txt
```

### 2. Train the ML Model (one-time)
```bash
python model.py
```
> Generates `claim_model.pkl`. Skip if already present.

### 3. Start the Flask API
```bash
python app.py
```
> Runs on **http://localhost:5001**
> Auto-trains the model if `claim_model.pkl` is missing.

### 4. Start the Next.js Frontend (in a separate terminal)
```bash
cd ../frontend
npm run dev
```
> Navigate to **http://localhost:3000/claim-intelligence**

## 🌐 API Reference

```json
{ "status": "ok", "service": "MediSync360 Insurance AI", "port": 5001 }
```

```json
{

  "disease": "Diabetes",
  "treatment_type": "Surgery",
  "company": "Star Health",
```

### `GET /companies`
Returns all supported insurers with key policy parameters.

### `POST /train`
Retrains the ML model on a fresh synthetic dataset.

| ICICI Lombard | 1% of SI/day  | 10%    | 2 years        | 20%                 |

## 🧠 ML Model Details

- **Algorithm:** Logistic Regression (scikit-learn Pipeline)
- **Features:** age, disease, treatment_type, company, sum_insured, waiting_completed, network_hospital, total_claimed
- **Encoding:** OneHotEncoder for categorical, StandardScaler for numerical
- **Training data:** 500 synthetic rows generated by `data_generator.py`
- **Accuracy:** ~67–72% (expected for rule-derived synthetic labels)
